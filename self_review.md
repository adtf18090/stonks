# 프로젝트 기획 계기
1. 뉴스를 보다 우연히 코스피 지수가 9000을 돌파한 것을 보았고, 주식을 검색해보니 삼성전자가 30만 원을 돌파했다는 사실을 알아챘음.
2. 예전에 8만 원에 샀다가 코로나 시즌을 지나고 8만 1000원에 팔았던 기억이 생각나서, 실시간으로 주식을 확인하고 분석할 수 있는 나만의 주식 검색 프로그램을 만들어보고 싶었음.
3. 그리하여 이 대시보드 프로그램을 제작하게 되었음.

# 자기평가

## 1. 잘했던 점 (문제 해결 및 성과):
- 객체지향 설계와 패키지화: 기존 단일 파일이었던 코드를 `BaseMarketFetcher` 부모 클래스와 `YFinanceFetcher` 자식 클래스로 분리하여 상속을 구현하고, 성공적으로 패키지화(.zip)하여 실무 적용 능력을 보여주었다.
- 트러블슈팅 (에러 해결):
  1) `customtkinter` UI와 `matplotlib`의 `tight_layout` 충돌로 발생하는 UserWarning을 `warnings.catch_warnings()`로 제어하여 터미널을 깔끔하게 유지했다.
  2) `yfinance`의 한계인 1주(15분봉) 차트 데이터 중간 누락(NaN) 현상을 `dropna()`를 활용해 전처리하여 차트가 끊기지 않게 완벽히 해결했다.
- 테스트 및 스타일 검증: pytest 9개 케이스를 엣지 케이스 포함하여 100% 통과했으며, pycodestyle의 79자 제한과 수많은 포매팅 에러를 해결하여 PEP 8 규칙을 완벽 준수했다.

## 2. 아쉬웠던 점 및 AI 도구 활용 내역: 
- AI 활용 내역 (제한적 허용 가이드 준수): 
  객체지향 파일 분리 구조를 잡을 때, 79자 제한(E501) 등 PEP 8 스타일 경고를 해결할 때, 그리고 GitHub 연동(환경변수, 터미널 오류) 과정에서 막히는 부분들을 AI(LLM)에게 질문하였다. AI가 제시한 `black`, `autopep8` 등의 포매터 사용법과 `# noqa: E501` 예외 처리 방법, 터미널 명령어 등의 답변을 바탕으로 에러 원인을 이해하고 코드를 직접 다듬어 적용했다.
- 아쉬웠던 점:
  AI 사용량이 다 되면 작업을 잠시 멈춰야 했다는 점, 그리고 VS Code 내부의 Git GUI 기능을 뒤늦게 활용하게 되면서 기존에 만들어둔 저장소를 폐기하고 새로 연결해야 했던 과정이 조금 아쉬웠다.

## 3. 다음에 개선하고 싶은 점:
- 뉴스 기사를 현재의 단순 RSS 피드 요약보다 조금 더 상세히(본문 내용 포함) 불러오는 파싱 기능을 추가해보고 싶다.
- 매수/매도 시뮬레이션(모의투자) 기능을 추가해 실제 주식 앱처럼 사용해보고 싶다.
- `yfinance`를 불러오는 방식 대신, 실제 금융권 앱이 사용하는 WebSocket 기반의 빠르고 정확한 실시간 API 연동 방식을 학습하여 적용해보고 싶다.
